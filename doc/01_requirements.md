# 功能需求文档（同步版）

> 更新时间：2026-03-06

## 1 核心需求

### 1.1 截图能力

必须支持三种模式入口：

- 全屏截图（已实现）
- 区域截图（已实现）
- 滚动截图（待实现）

区域截图要求：

1. 进入全屏遮罩层
2. 鼠标拖拽选区
3. 显示选区尺寸提示
4. 支持 Enter 确认 / ESC 取消
5. 支持覆盖层工具条确认

### 1.2 结果处理

- 可预览截图
- 可保存 PNG
- 可复制到剪贴板
- 失败必须有提示

### 1.3 主界面交互

主界面所有按钮需满足：

- 已实现功能：触发真实业务
- 未实现功能：必须给出明确提示（不能无响应）

---

## 2 UI 与交互需求

### 2.1 主窗口

- 卡片化布局
- 左侧导航 + 顶部操作区 + 中央内容区 + 右侧设置区
- 分段模式按钮三等分占满宽度
- 选中模式有明显视觉反馈

### 2.2 覆盖层

- 半透明暗色遮罩
- 选区高亮边框
- 选区尺寸文本
- 选区后工具条自动定位（上方优先，下方兜底）

工具条按钮：

- 编辑：画笔、矩形、圆形、马赛克、文字（待实现）
- 操作：复制、保存、取消、确认（已接入）

### 2.3 提示系统

- 使用 `ShowTip` 展示短时提示
- 提示文案统一中文

---

## 3 配置需求（近期）

- 热键配置
- 导出格式配置（PNG/JPG）
- 自动保存开关与目录

---

## 4 非功能需求

### 性能

- 截图延迟目标：< 100ms（常规分辨率）
- UI 响应目标：< 16ms

### 资源占用

- 常驻内存目标：< 200MB（含历史缩略图缓存）

### 兼容

- Windows 10+
- Qt6 Widgets

### 可维护性

- 样式统一由 QSS 管理
- 图标优先 SVG 资源
- 模块职责清晰，便于后续拆分 Service
