# 需求文档（同步版）

> 更新时间：2026-03-06

## 1. 核心需求

### 1.1 截图能力

必须支持：

- 全屏截图
- 区域截图
- 长截图（集成在区域模式）

说明：不再保留“独立滚动模式”作为可用业务入口，长截图通过区域选区后的工具条进入。

### 1.2 结果处理

- 截图结果可预览
- 支持保存 PNG
- 支持复制到剪贴板
- 失败必须提示

### 1.3 多屏支持

- 全屏模式支持选择目标显示器
- 区域模式支持多屏/跨屏选区

## 2. 交互需求

### 2.1 主界面

- 保持非透明主窗口
- 采用卡片化布局和统一 QSS
- 模式切换以“全屏 / 区域”为主

### 2.2 覆盖层

- 半透明遮罩
- 拖拽选区
- 轻微蓝色边框（仅选区视觉提示）
- 最终成图不得包含蓝色边框
- 选区后显示工具条

### 2.3 长截图流程

1. 区域选区完成
2. 点击工具条“长截图”进入长截图状态
3. 用户滚轮滚动目标内容
4. 系统按帧增量拼接
5. 点击保存或确认（复制）完成输出

## 3. 非功能需求

- 代码统一 UTF-8
- 编译过程避免乱码导致的字符串截断
- 模块职责清晰，便于后续扩展服务层
