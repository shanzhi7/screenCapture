# 需求文档（同步版）

> 更新时间：2026-03-07

## 1. 核心需求

### 1.1 截图能力

必须支持：

- 全屏截图
- 区域截图
- 长截图（集成在区域模式）

说明：独立滚动模式不再作为主路径入口，长截图统一通过区域选区后的工具条触发。

### 1.2 结果处理

- 截图结果可预览
- 支持保存 PNG
- 支持复制到剪贴板
- 关键失败必须有提示

### 1.3 多屏支持

- 全屏模式可选择目标显示器
- 区域模式支持跨屏选区
- 全局热键触发时可在任意显示器启动选区截图

## 2. 交互需求

### 2.1 主界面

- 主窗口必须保持可见（禁止整体透明）
- UI 使用统一 QSS 风格（HyperOS 方向）
- 模式切换聚焦“全屏 / 区域”

### 2.2 覆盖层

- 半透明遮罩
- 拖拽选区
- 选区交互态保留轻微蓝色描边
- 最终截图成图不得包含蓝色描边
- 选区后显示工具条（保存/取消/确认/长截图/编辑入口）

### 2.3 长截图流程

1. 区域选区完成
2. 点击工具条“长截图”进入长截图状态
3. 用户滚轮滚动目标内容
4. 系统按帧增量拼接
5. 点击保存或确认（复制）完成输出

### 2.4 应用生命周期

- 点击窗口关闭按钮需弹出关闭行为对话框
- 用户可选择：隐藏到系统托盘 / 退出应用 / 取消
- 退出应用后托盘图标必须消失

## 3. 非功能需求

- 代码文件统一 UTF-8 编码
- 避免乱码导致字符串截断或编译异常
- 模块职责清晰，可继续拆分服务层
- 文档与代码状态保持同步
