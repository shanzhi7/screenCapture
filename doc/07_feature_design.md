# 功能设计文档（同步版）

> 更新时间：2026-03-14

## 1. 已实现功能

### 1.1 全屏截图

- 支持多屏环境选择目标显示器
- 抓图前临时隐藏主窗口，避免截入工具自身
- 结果自动复制到剪贴板
- 自动保存开启时，结果自动保存到用户配置目录（不可用时回退默认目录）

### 1.2 区域截图

- 覆盖层拖拽选区
- 确认（✓）后自动复制到剪贴板
- 保存（保存按钮）支持截图保存
- 支持 `Esc` / `Enter` 快捷操作
- 支持鼠标右键取消（等价 `Esc` / `✕`）
- 支持画笔 / 矩形 / 圆形 / 马赛克 / 文字标注
- 支持工具条按钮与 `Ctrl+Z` / `Ctrl+Y` 撤销重做

### 1.3 长截图（区域模式内）

- 启动方式：区域选区 -> 工具条“长截图”
- 滚轮触发抓帧
- 使用 `LongCaptureSessionController` 协调滚动 / 观测 / 拼接流水线
- 拼接链路包含 `ScrollDispatcher`、`MotionDetector`、`StableFrameCollector`、`OverlapMatcher`、`StitchComposer`
- 结果支持复制与保存（手动）
- 长截图过程中同样支持鼠标右键取消

### 1.4 历史记录（第一阶段）

- 新增 `CaptureHistoryManager` 持久化历史截图
- 新截图自动写入本地历史目录
- 主界面最近截图区域显示真实历史缩略图
- 点击历史缩略图可回显到预览区

### 1.5 输出格式配置

- 支持 PNG / JPG 一键切换
- 格式按钮文案实时反映当前格式
- 保存文件默认扩展名随格式变化
- 当前格式通过 `SettingsService` 持久化到 `QSettings`

### 1.6 自动保存逻辑

- 默认目录：系统图片目录下 `轻影截图`
- 支持用户自定义自动保存目录：`capture/auto_save_dir`
- 文件名冲突自动重命名（追加序号）
- 全屏截图：自动复制 + 自动保存（受开关控制）
- 区域截图：仅点击工具条“保存”时触发自动保存（受开关控制）
- 长截图：保持手动保存流程
- 自动保存目录不可写或创建失败时回退默认目录并提示

### 1.7 热键系统

- Windows 全局热键由 `GlobalHotkeyManager` 统一管理
- 热键持久化键：`capture/hotkey_sequence`
- 热键修改后立即生效
- 非法组合（无修饰键）会被拒绝
- 注册失败时回滚到上一次有效热键并提示原因

### 1.8 设置入口与按钮职责

- `btnTopSettings`：打开截图设置对话框
- `sideConfigButton`：打开截图设置对话框（侧栏入口）
- `btnHotkeySetting`：打开截图设置（定位热键配置）
- `sideBottomButton`：打开关于 / 反馈对话框

### 1.9 配置命名升级与迁移

- 发布命名空间：`QingYing/LightShadowCapture`
- 首次启动自动迁移旧命名空间：`shanzhi/screenCapture`
- 迁移策略：新值优先，不覆盖新命名空间已有键
- 迁移标记：`meta/settings_migrated_from_legacy`

### 1.10 提示与托盘一致性

- `ShowTip` + `TipPresenter` 统一短时提示（鼠标优先，越界回退右下角）
- 主窗口隐藏到托盘后，截图完成不自动恢复主窗口
- 托盘隐藏状态下手动保存仅弹保存对话框，不恢复主窗口

### 1.11 视觉与图标

- 主界面和弹窗采用统一项目风格 QSS
- 应用窗口图标和 EXE 图标已接入 `icon.ico`

## 2. 部分实现功能

- 长截图已接入实验链路，当前仓库默认构建开启，但复杂动态内容与更多 Windows 场景仍需继续验收
- 历史分页、删除、筛选等高级能力未实现
